# 🌿 PlantCare PRO — Инструкция пользователя

## Как запустить проект

1. **Скопируйте `.env.example` в `.env`** и впишите свои ключи API:
   ```
   cp .env.example .env
   ```
   Откройте `.env` и замените значения на свои:
   - `TREFLE_API_KEY` — получить на https://trefle.io (бесплатно)
   - `PLANT_ID_API_KEY` — получить на https://web.plant.id (бесплатно, 1000 запросов/мес)

2. **Установите зависимости** (в терминале, в папке проекта):
   ```
   npm install
   ```

3. **Запустите сервер**:
   ```
   node server.js
   ```

4. **Откройте в браузере**: http://localhost:3000

> ⚠️ Через Live Server (порт 5500) POST-запросы работать НЕ будут. Запуск только через `node server.js`.

## Описание всех функций

### 📚 Справочник растений
- Карточки растений с подробной информацией (полив, освещение, пересадка, ядовитость)
- **Поиск** по названию (строка поиска над карточками)
- **Фильтры по категориям**: Все / Комнатные / Ядовитые / Колючие / Цветущие / Съедобные / Лекарственные
- **Кнопка «🔍 Найти в интернете»** — открывает Google с названием растения
- **Загрузка своего фото** — если у растения нет фотографии (SVG-заглушка), на карточке появляется кнопка «📤 Загрузить свою фотографию»
- Добавление в коллекцию, добавление в избранное (❤️), переход в чат

### 🌱 Моя коллекция
- Список ваших растений с таймерами полива и пересадки
- Кнопки «Полил(а)» и «Пересадил(а)» для отметки действий
- **Кнопка «🔄 Предложить обмен»** — открывает форму создания предложения обмена

### ❤️ Избранное
- Растения, которые вы пометили сердечком в справочнике

### 💬 Обсуждения (чат по растениям)
- Выберите растение → откроется общий чат, где все пользователи обсуждают уход
- Сообщения обновляются автоматически каждые 5 секунд

### 💬 Общение (личный мессенджер)
- **Поиск пользователей** по никнейму (кнопка «🔍 Найти пользователя»)
- **Личные чаты** (1 на 1) — нажмите «Написать» рядом с найденным пользователем
- **Групповые чаты** — кнопка «👥 Создать группу», выберите участников
- **Уведомления**: красный кружок с числом непрочитанных, звук, браузерные всплывашки
- **Меню чата** (три точки ⋮):
  - 🚫 Заблокировать пользователя (не сможет отправлять вам сообщения)
  - 📌 Закрепить чат (всегда вверху списка)
  - 🔕 Отключить уведомления (без звука и всплывашек)
  - ✏️ Переименовать группу (только для групповых чатов)
  - 🗑️ Удалить чат (с подтверждением)
- Сообщения обновляются автоматически (поллинг каждые 5 секунд)

### 🔄 Обмен растениями
- **Предложить обмен** — из карточки в «Моей коллекции» или из вкладки «Обмен»
- Укажите растение, получателя (или оставьте пустым для открытого предложения) и комментарий
- **Под-вкладки**: Все предложения / Мои / Адресованные мне
- **Статусы**: Ожидание → Принято / Отклонено
- Можно отменить своё предложение или принять/отклонить чужое

### 📷 Распознать растение по фото
- Загрузите или перетащите фото в зону загрузки
- Сервис Plant.id определит вид растения и покажет точность
- Если растение найдено в справочнике — кнопка «Открыть карточку»
- Если не найдено — ссылка для поиска в интернете
- **История**: последние 10 распознаваний сохраняются в браузере

### 🌙 Смена темы оформления
- Кнопка ☀️/🌙 в заголовке сайдбара
- Тёмная и светлая тема с плавной анимацией
- Зелёные акценты сохраняются в обеих темах
- Выбор запоминается между сессиями

## Автозаполнение справочника из Trefle API

```
node seed-plants.js
```

Скрипт загрузит до 40 растений из Trefle и добавит их в `database.json`. Безопасно запускать повторно — дубликаты не создаются. После сидирования можно запустить `node migrate-categories.js` для автоматической категоризации.

## FAQ для преподавателя

**В: Как работает авторизация?**
О: Логин/пароль хранятся в `database.json` в открытом виде (это учебный проект, не production). При входе сервер ищет совпадение, при регистрации — проверяет уникальность никнейма.

**В: Где хранятся данные?**
О: Всё в одном файле `database.json` — это проще для понимания, чем настоящая БД. Разделы: `catalog` (справочник), `users` (пользователи), `collections` (коллекции), `favorites` (избранное), `chats` (чат по растениям), `messages` (личный мессенджер), `tradeOffers` (обмен), `groups` (группы), `blockedUsers`/`pinnedChats`/`mutedChats` (настройки чатов).

**В: Почему ключи API в .env, а не в коде?**
О: Безопасность. Файлы .js, которые загружаются в браузер, видны всем пользователям сайта. Если положить ключ в такой файл — его легко украсть. Поэтому ключи лежат на сервере в .env, а фронтенд обращается к своим собственным роутам (/api/recognize, /api/trefle/search), которые уже на сервере подставляют ключ.

**В: Как работает распознавание растений?**
О: Фронтенд кодирует фото в base64 → отправляет на свой сервер (/api/recognize) → сервер обращается к Plant.id API v3 с секретным ключом → получает список возможных видов с вероятностями → возвращает результат фронтенду.

**В: Как работает мессенджер без WebSocket?**
О: Через поллинг (polling) — клиент каждые 5 секунд спрашивает сервер «есть ли новые сообщения?». Это проще WebSocket, но создаёт небольшую задержку (до 5 секунд).
